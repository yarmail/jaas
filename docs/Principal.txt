Principal — это интерфейс, который представляет собой
абстракцию для идентификации субъекта. Субъектом может
быть пользователь, группа пользователей или система.
Интерфейс предоставляет методы для получения имени субъекта
и проверки его прав доступа.

Основная задача Principal — представлять
уникального пользователя или систему в приложении.

1. Система запрашивает учетные данные
(например, имя пользователя и пароль).
2. Эти учетные данные проверяются с помощью модуля входа (LoginModule).
3. Если аутентификация успешна, создается экземпляр класса,
реализующего интерфейс Principal, который представляет этого пользователя.
4. Объект Principal добавляется в объект Subject,
который содержит информацию о текущем пользователе.
5. В дальнейшем приложение может использовать этот объект
для проверки прав доступа к ресурсам на основе идентификатора пользователя.
